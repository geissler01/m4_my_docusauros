# Documentación de Actividades

Bienvenido a mi espacio de documentación académica.

Este sitio ha sido creado con el propósito de organizar, estructurar y evidenciar las actividades desarrolladas durante el módulo 4 de bases de datos. Aquí registro procesos, ejercicios prácticos y avances en mi formación.

---

## Objetivo de esta documentación

El objetivo principal es:

- Consolidar el aprendizaje mediante la escritura técnica usando Markdawn.
- Mantener un registro organizado de proyectos y prácticas en clases.
- Documentar procesos paso a paso para futura referencia.
- Construir evidencia de desarrollo profesional y técnico.

---

## Metodología de trabajo

Cada actividad documentada incluye:

- Contexto del ejercicio o proyecto.
- Desarrollo técnico (código, estructura, lógica).
- Herramientas utilizadas.
- Resultados obtenidos.
- Reflexión o conclusiones.

El enfoque es técnico, estructurado y progresivo, permitiendo visualizar la evolución del aprendizaje a lo largo del tiempo.

---

## Contenido del sitio

En este espacio se documentarán actividades relacionadas con:

- Manejo de bases de datos
- Desarrollo web (Frontend y Backend)
- Estructuración de proyectos
- Buenas prácticas de programación
- Fundamentos técnicos aplicados en clase

Cada sección está organizada para facilitar la navegación y comprensión del proceso.

---

##  Propósito profesional

Más allá del ámbito académico, esta documentación funciona como:

- Portafolio técnico
- Evidencia de habilidades prácticas
- Registro de crecimiento profesional
- Material de consulta futura

Documentar es una forma de aprender dos veces: al hacer y al explicar.

---

Gracias por visitar este espacio de aprendizaje.

# Tutorial Intro

Let's discover **Docusaurus in less than 5 minutes**.

## Getting Started

Get started by **creating a new site**.

Or **try Docusaurus immediately** with **[docusaurus.new](https://docusaurus.new)**.

### What you'll need

- [Node.js](https://nodejs.org/en/download/) version 20.0 or above:
  - When installing Node.js, you are recommended to check all checkboxes related to dependencies.

## Generate a new site

Generate a new Docusaurus site using the **classic template**.

The classic template will automatically be added to your project after you run the command:

```bash
npm init docusaurus@latest my-website classic
```

You can type this command into Command Prompt, Powershell, Terminal, or any other integrated terminal of your code editor.

The command also installs all necessary dependencies you need to run Docusaurus.

## Start your site

Run the development server:

```bash
cd my-website
npm run start
```

The `cd` command changes the directory you're working with. In order to work with your newly created Docusaurus site, you'll need to navigate the terminal there.

The `npm run start` command builds your website locally and serves it through a development server, ready for you to view at http://localhost:3000/.

Open `docs/intro.md` (this page) and edit some lines: the site **reloads automatically** and displays your changes. -->
